# usePageContent

usePageContext accepts the props from getServerSideProps or getStaticProps to create and maintain the view of Page Content.

## Input Props

1. initialContent - the content in raw MD/MDX format.
2. initialContext - the context within which to run (from site.config.js).


## Output Props

1. pageContent - the processed MDX content.
2. content - the raw content as loaded (used in the Editor)
3. *contentSource - the source of the content ['etherpad', 'github'] (is this needed?)*
4. menuStructure - the appropriate menu structure based off the collection as a root.
5. context - the current context

## Handlers

1. handleContentChange(uri, Bollean(relative)) - processes in app content navigation. `relative` signifies that it's related content to append to the url as `?page=``
2. handlePageReset - resets the page to the baseline initialContext
3. *setEditMode - probably not needed*
4. *frontMatterCallback - updates the frontmatter (is this needed?)*
